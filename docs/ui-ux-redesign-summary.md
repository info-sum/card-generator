# SNS 카드 뉴스 생성기 UI/UX 개편용 정리 요약

작성 기준일: 2026-03-20

## 1. 서비스 한 줄 정의

사용자가 휴대폰 사진첩에서 이미지 3~5장을 선택하면, 자동으로 카드뉴스 초안을 만들고 텍스트만 수정해서 SNS 카드뉴스 또는 앱스토어 소개 이미지를 PNG로 저장할 수 있는 제작 도구입니다.

## 2. 현재 핵심 가치

- 이미지를 여러 장 올리면 카드형 흐름이 바로 시작됨
- 장별 카피를 빠르게 수정할 수 있음
- SNS 카드뉴스와 앱스토어 소개 이미지를 같은 입력으로 만들 수 있음
- 미리보기와 저장이 한 화면 안에서 이어짐
- 초안 자동 저장과 복구가 가능함

## 3. 현재 기능 범위

### 입력

- 사진첩에서 이미지 여러 장 선택
- 이미지 최대 5장
- 권장 장수 3~5장
- 이미지 파일만 지원

### 편집

- 서비스 이름 수정
- 메인 메시지 수정
- 출력 모드 전환
  - SNS 카드뉴스
  - 앱스토어 소개 이미지
- 해상도 프리셋 선택
  - SNS 1080x1080
  - SNS 1080x1350
  - SNS 1080x1920
  - App Store 1320x2868
  - App Store 1284x2778
  - App Store 1125x2436
- 테마 선택
  - Ember
  - Tide
  - Graphite
- 슬라이드별 텍스트 수정
  - 상단 라벨
  - 보조 배지
  - 헤드라인
  - 설명
- 슬라이드 순서 이동
- 슬라이드 삭제

### 출력

- 개별 슬라이드 PNG 저장
- 전체 슬라이드 PNG 저장
- 파일명 자동 생성
- 앱인토스 런타임에서는 기기 저장
- 브라우저에서는 다운로드

### 저장

- 초안 자동 저장
- 초안 수동 저장
- 이전 초안 자동 복구
- 새 프로젝트 초기화

## 4. 현재 사용자 흐름

1. 앱 진입
2. 서비스 이름, 메인 메시지, 출력 모드, 해상도, 테마 선택
3. 사진첩에서 이미지 3~5장 선택
4. 이미지 수만큼 슬라이드 초안 생성
5. 각 장의 텍스트 수정
6. 순서 조정 또는 삭제
7. 하단 미리보기 확인
8. 개별 저장 또는 전체 저장

## 5. 현재 화면 구조

### 상단 히어로

- 앱 로고
- 앱 이름
- 앱 설명
- 현재 장수 표시
- 현재 런타임 상태 표시
  - AppsInToss 연결됨
  - 브라우저 미리보기 모드

### 1구역: 프로젝트 설정 패널

- 서비스 이름 입력
- 메인 메시지 입력
- 출력 모드 칩
- 해상도 칩
- 테마 칩
- 사진첩에서 가져오기 버튼

### 2구역: 현재 상태 패널

- 작업 상태 메시지
- 안내 문구
- 제약 조건 체크리스트
- 전체 저장
- 초안 저장
- 새 프로젝트

### 3구역: 슬라이드 편집 영역

- 업로드한 이미지 썸네일
- 순번 표시
- 슬라이드별 텍스트 입력 폼
- 위로 / 아래로 / 삭제 / 이 장 저장 버튼

### 4구역: 미리보기 영역

- 실제 출력 비율 축소 미리보기
- 각 슬라이드 카드 표시
- 현재 프리셋 기준 출력 결과 확인

## 6. 현재 자동화 규칙

### 슬라이드 생성 규칙

- 이미지를 추가하면 자동으로 슬라이드 데이터 생성
- 출력 모드에 따라 기본 카피 세트가 달라짐
- SNS 카드뉴스 모드는 훅, 메시지, 전개, 사용 장면, 마무리 구조를 기본으로 가짐
- 앱스토어 소개 이미지 모드는 메인 가치, 차별 포인트, 핵심 기능, 신뢰 요소, 마지막 장 구조를 기본으로 가짐

### 저장 규칙

- 편집 데이터는 0.5초 단위로 자동 저장
- 저장 키는 현재 초안 기준 하나만 유지
- 파일명은 브랜드명 + 모드 + 순번 조합으로 생성

## 7. 현재 디자인 시스템 요약

### 기본 톤

- 라이트 모드 기반
- 밝은 크림 계열 배경 + 흰색 패널
- 오렌지 계열 포인트 컬러
- 라운드가 큰 카드형 UI
- 그라데이션 강조 버튼 사용

### 기본 브랜드 팔레트

현재 기본 테마는 `Ember`입니다. 개편 시 색상만 유지하려면 아래 값을 기준으로 가져가면 됩니다.

- 메인 포인트: `#dd5e31`
- 포인트 보조: `#ffcf9a`
- 본문 텍스트: `#241914`
- 보조 텍스트: `rgba(76, 54, 42, 0.72)`
- 페이지 배경:
  - `linear-gradient(180deg, rgba(255, 248, 241, 0.98) 0%, rgba(255, 240, 229, 0.98) 100%)`
- 패널 배경: `rgba(255, 255, 255, 0.96)`
- 패널 보더: `rgba(184, 129, 96, 0.18)`
- 패널 그림자: `0 24px 48px rgba(130, 85, 57, 0.12)`

### 결과물용 비주얼

- SNS 카드뉴스는 강한 다크 배경 그라데이션 위에 이미지와 텍스트가 올라감
- 앱스토어 소개 이미지는 라이트에서 다크로 내려가는 배경과 아이폰 목업을 사용함
- 결과물 자체의 스타일과 앱 편집 UI의 스타일은 분리되어 있음

## 8. 현재 UX 장점

- 기능이 단순해서 진입 장벽이 낮음
- 입력부터 저장까지 한 화면에서 끝남
- 모드 전환이 쉬움
- 미리보기와 결과물이 크게 다르지 않음
- 초안 저장이 자동이라 재진입에 유리함

## 9. 현재 UX 한계

- 화면이 길고 세로로 많이 이어져서 사용자가 지금 어느 단계인지 직관적으로 알기 어려움
- 사진 업로드가 핵심 시작점인데, 시작 행동이 상대적으로 약하게 보임
- 프로젝트 설정, 편집, 미리보기가 분리되어 있어서 시선 이동이 많음
- 슬라이드가 많아지면 편집 폼이 길어져 피로감이 커짐
- 각 슬라이드의 전체 흐름보다 개별 폼 입력에 집중되기 쉬움
- 정렬 방식이 버튼 기반이라 재배치가 다소 느림
- 테마 선택은 가능하지만 결과 차이를 썸네일로 비교하기 어려움
- 저장 버튼이 상태 패널 안에 있어 최종 행동의 존재감이 조금 약함

## 10. 개편 시 유지해야 할 제약

- 색상 톤은 유지
- 이미지 기반 입력 유지
- 기본 장수 제한은 5장 유지
- SNS 카드뉴스 / 앱스토어 소개 이미지 두 모드 유지
- PNG 저장 유지
- 초안 저장 / 복구 유지
- 결과물 렌더링은 현재 DOM 기반 내보내기 구조를 유지하는 편이 안전
- 앱스토어 모드는 아이폰 목업과 한 장 한 메시지 원칙 유지

## 11. 개편 방향 제안

### 방향 1: 단계형 워크플로우로 재구성

- 현재 긴 단일 페이지를 `설정 > 이미지 선택 > 장별 편집 > 저장` 흐름으로 명확히 나누기
- 상단에 진행 단계 표시를 두면 사용자가 현재 위치를 바로 이해할 수 있음

### 방향 2: 편집과 미리보기를 한 화면에서 동시에 보기

- 좌측은 슬라이드 목록과 편집
- 우측은 실시간 미리보기
- 저장 버튼은 하단 고정 액션 바로 분리

### 방향 3: 슬라이드 목록 중심 UX로 전환

- 현재는 긴 입력 폼 중심 구조
- 개편 시에는 슬라이드 썸네일 리스트를 먼저 보여주고, 선택된 장만 상세 편집하는 구조가 더 효율적임

### 방향 4: 업로드 시작 경험 강화

- 첫 화면에서 가장 먼저 보여야 할 행동은 `사진첩에서 가져오기`
- 빈 상태에서는 설명보다 업로드 CTA가 더 강하게 보여야 함

### 방향 5: 저장 액션 강조

- 전체 저장은 항상 보이는 고정 영역으로 분리
- 현재 상태 패널 안에 섞여 있는 구조보다 더 명확한 마무리 액션이 필요함

## 12. 추천 IA 초안

### A안: 2단 편집 스튜디오

- 상단:
  - 브랜드
  - 모드 전환
  - 해상도
  - 테마
- 좌측:
  - 사진 업로드
  - 슬라이드 리스트
  - 순서 변경
- 중앙:
  - 선택된 슬라이드 편집
- 우측:
  - 실시간 미리보기
- 하단 고정:
  - 전체 저장
  - 현재 상태

### B안: 단계형 제작 플로우

- 1단계: 이미지 선택
- 2단계: 흐름 정리
- 3단계: 카피 수정
- 4단계: 저장

이 구조는 처음 사용하는 사람에게 더 친절합니다.

## 13. 개편 우선순위

### 1순위

- 정보 구조 재배치
- 업로드 CTA 강화
- 편집과 미리보기 동시 노출
- 저장 액션 재배치

### 2순위

- 슬라이드 선택형 편집 구조
- 순서 변경 UX 개선
- 테마 미리보기 강화

### 3순위

- 텍스트 길이 가이드
- 자동 표지 / 마지막 장 추천
- 템플릿 프리셋 시각화

## 14. 한 줄 결론

이 프로젝트는 기능 자체보다 `긴 단일 페이지 편집기`라는 현재 구조가 UX 한계의 핵심입니다. 색상과 브랜드 톤은 유지하되, `업로드 시작 > 슬라이드 흐름 편집 > 실시간 미리보기 > 저장`이 더 짧고 명확하게 이어지는 구조로 재설계하는 것이 가장 효과적입니다.
